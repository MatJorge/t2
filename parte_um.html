Assicronismo

O assnicronismo é uma ferramenta em JavaScript que permite que as respostas de operações com duração mais longa sejam executadas de forma individual e independente, sem comprometer a execução do código.



AJAX

Permite que o navegador receba informações de um servidor atualizando a página da web sem a necessidade de atualizá-la. Vantagens: Ampla compatibilidade com navegadores. Flexibilidade em formatos de dados (XML, JSON, etc.).Desvantagens: Código mais verboso. Difícil de lidar com múltiplas requisições.

Promises

São objetos que representam conclusão e falha de uma operação assíncrona. Vantagens: Melhora a legibilidade do código. Permite encadeamento. Desvantagens: Pode ser confuso para iniciantes. Não trata automaticamente erros em encadeamentos.

Fetch API

É uma interface mais limpa e moderna para as requisições assíncronas, baseada em promises. Vantagens: Sintaxe mais limpa e moderna. Baseada em Promises, o que facilita o encadeamento. Desvantagens: Não é suportada em alguns navegadores antigos (ex: Internet Explorer).

Async/Await

São palavras chave que facilitam o uso de promises. Vantagens: Sintaxe mais simples e parecida com código síncrono. Facilita o tratamento de erros com try/catch. Desvantagens: Necessita de suporte do navegador.Não pode ser usado em funções que não são marcadas como async.

Outros Temas



Hoisting

É a utilização de uma variável antes de declará-la, no seu contexto.

Arrow Functions

São formas concisa de escrever uma funçao, herdando o "this" do contexto que foram definidas

Desestruturação

É uma sintaxe que permite extrair valores de uma Array ou de Objetos.

Closure

Closure é uma função que "lembra" seu escopo léxico mesmo quando a função é executada fora desse escopo. Isso permite que uma função acesse variáveis de seu escopo externo mesmo após a execução do contexto onde foi criada.
